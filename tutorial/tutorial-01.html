1.2 Insert and list the data

So far, so good. However, the interface at the moment does not do anything. Entering a new value and clicking on the button will not change anything. So how do we achieve that? Before we start adding functionality we need to think about what data we need to store. For this first version we only need a collection of all the wishes (let’s call it WishList) where each item in the collection will have a name. Explain the example

Let’s understand better what the new annotations represent:

  • with data-value annotation we have declared a reference label itemName for the value entered into the input box. This allows us to reference that value in other places (as we will see soon).
  • with data-click annotation we declare what should happen when the button is clicked. In this case insert({name:itemName}) specifies that a new object should be inserted into a collection and that its name should be the value from the input box (referenced by itemName). But what collection does it get inserted in?
  • well that’s what insert-into="WishList" is for. This annotations specifies the collection where the data from the "insert(...)" operation is supposed to go. And this is our desired WishList collection.
  • finally, we replaced the two static list items (<li>..</li>) we had before with one annotated with a from annotation where we define the query to get the data. This way we have defined a template that will be duplicated for each element in the retrieved data. In our case we said WishList item meaning we want to get the data from WishList and item label to reference the individual item (similar to how we used itemName earlier). We can see how the item is used inside the {{item.name}} where we just want to display the item’s name. Double curly brackets are used whenever we want to just display data we have somehow retrieved or created through annotations.

Now if you have any experience with web you may think: “This is a nice trick, but if I close my browser the data will be lost”. Well, go ahead and try . Still there? If you opened the same browser your data should still be there. The catch is in “the same browser”. Endev use browser storage to dynamically create your data structure and store your data. It is similar to how cookies work and that is why the data is only local and in another browser or another computer you would not see it. But we will solve this issue soon.

Let’s recap what we have learned in this part of the tutorial:

  • Endev supports interactivity with data through HTML annotations
  • from annotation can be used to retrieve data from arbitrary collections (if one does not exist it will be created for you on-the-fly) and combined with auto-update also saves any changes that the user does to the data through the interface
  • data-clickannotations in combination with insert(...), update(...) or delete(...) statements can be used to do exactly that, insert, update or delete the data
  • data-value annotation is used to give as a reference to values in input elements
  • The data has, for now, been only stored in the browser
